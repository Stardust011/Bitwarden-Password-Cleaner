# Bitwarden-Password-Cleaner

Bitwarden Password Cleaner 是一个用于清理和优化 Bitwarden 密码管理器导出文件的 Python 脚本。它可以删除重复条目，检查 URL 的有效性，并将清理后的数据导出到新的 JSON 文件中。

## 功能

Bitwarden Password Cleaner 提供以下功能，帮助您清理和优化 Bitwarden 密码管理器导出文件：
- **删除重复条目**：自动识别并删除基于名称、URI、用户名和密码的重复条目。
- **URL 验证**：通过网络访问与DNS解析检查 Bitwarden 条目中的 URL 的有效性。如果 URL 不可达，则从条目的 URIs 列表中删除。
- **优化导出**：导出一个清理和优化后的 Bitwarden 导出 JSON 文件，删除重复条目和无效 URL。
- **实时输出**：实时显示进度，显示正在处理的项目和删除原因（如果有）。
- **详细的删除报告**：生成一个单独的 JSON 文件，包含有关删除项目的信息，包括删除原因，供您参考。
- **用户友好**：设计易于使用，提供清晰的说明和可定制的设置，以适应您的偏好。

## 先决条件

在开始之前，请确保您满足以下要求：

- 系统上安装了 Python 3.10 或更高版本。
- 一个 Bitwarden 密码管理器导出 JSON 文件。

## 快速开始

1. 将此存储库克隆到本地计算机：

   ```bash
   git clone https://github.com/Stardust011/Bitwarden-Password-Cleaner.git
   ```

2. 导航到项目目录：

   ```bash
   cd Bitwarden-Password-Cleaner
   ```

3. 安装所需的 Python 包：

   ```bash
   pip install rich requests tldextract
   ```

## 使用方法

要使用 Bitwarden Password Cleaner，请按照以下步骤操作：

1. 将您的 Bitwarden 导出 JSON 文件放在项目目录中。如您不想修改代码，可将文件重命名为`bitwarden_export.json`

2. (可选)打开 `bitwardenCleaner.py` 脚本，根据您的偏好配置文件名和其他设置。

3. 运行脚本：

   ```bash
   python BitwardenCleaner.py
   ```

4. 脚本将处理输入文件并生成三个输出文件：一个包含清理后的数据，一个包含删除的项目，还有一个是待您人工检查的项目，一般是账号或密码缺省。

## ⚠️ 高危操作警告

**在使用本工具之前，请务必仔细阅读以下高危操作说明。本工具会修改您的密码数据，使用不当可能导致数据丢失。**

### 🔴 关键风险

1. **数据永久删除风险**
   - 脚本会自动删除被认定为"重复"的密码条目
   - 删除的条目仅保存在 `*_deleted.json` 文件中，原始文件不会被修改
   - **建议**: 运行脚本前，请务必备份原始 `bitwarden_export.json` 文件

2. **URL 验证网络请求**
   - 脚本会对每个 URL 进行网络可达性测试（HTTP HEAD 请求）
   - 会向第三方 DNS 服务器（Cloudflare 1.1.1.1）发送 DNS 查询请求
   - 这可能会：
     - 暴露您访问的网站域名信息
     - 在某些网络环境下失败（防火墙、代理等）
     - 触发某些网站的安全监控
   - **建议**: 在安全的网络环境中运行，考虑使用 VPN

3. **敏感数据暴露**
   - 所有输出文件（`*_output.json`, `*_deleted.json`, `*_check.json`）以明文 JSON 格式存储
   - 这些文件包含您的用户名和密码
   - **建议**: 
     - 处理完成后立即加密或删除这些文件
     - 确保文件存储在安全的位置
     - 不要将这些文件上传到云端或公共位置

4. **文件覆盖风险**
   - 如果输出文件已存在，脚本会直接覆盖，不会提示
   - **建议**: 首次运行前检查是否存在同名文件

5. **URL 重定向跟踪**
   - 脚本会跟随 HTTP 重定向并更新 URL
   - 这可能导致原始 URL 被替换为不同的域名
   - **建议**: 运行后检查 `*_check.json` 文件，人工验证修改的 URL

### 🟡 次要风险

6. **依赖外部服务**
   - DNS 查询依赖 Cloudflare（1.1.1.1）的可用性
   - 如果该服务不可用或被阻止，部分验证功能会失败
   - DNS 查询失败会导致该 URL 被认为无效并删除

7. **不完整的重复检测**
   - 重复检测基于：网站名称、用户名和密码完全匹配
   - 如果两个条目的密码不同但实际上是同一账户，不会被检测为重复
   - 可能错过某些应该合并的条目

8. **超时设置**
   - 网络请求超时设置为 5 秒
   - 在网络较慢的环境下，可能会误判有效 URL 为不可达

### 🔵 安全建议

- ✅ **运行前必做**:
  1. 完整备份原始 `bitwarden_export.json` 文件
  2. 在测试环境中先用少量数据测试
  3. 确保在安全的网络环境中运行

- ✅ **运行后必做**:
  1. 仔细检查 `*_check.json` 文件中需要人工确认的项目
  2. 验证 `*_deleted.json` 中的删除项目是否正确
  3. 测试导入 `*_output.json` 到 Bitwarden 前，先备份现有数据
  4. 处理完成后，安全删除所有包含密码的 JSON 文件

- ✅ **隐私保护**:
  1. 不要在共享或公共网络环境中运行
  2. 运行期间注意屏幕内容可能包含敏感信息
  3. 考虑在离线或隔离环境中处理敏感数据
